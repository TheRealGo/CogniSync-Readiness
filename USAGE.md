# 使い方ガイド

## 必要なもの

- **[Nix](https://nixos.org/download/)** （パッケージマネージャー）
- **[direnv](https://direnv.net/docs/installation.html)** （推奨 / 自動で開発環境を読み込む）

Nix をインストール済みであれば、Node.js や pnpm を個別にインストールする必要はありません。`flake.nix` が必要なツールを自動で提供します。

> direnv を使わない場合は、各コマンドの前に `nix develop -c` を付けるか、`nix develop` でシェルに入ってから実行してください。

## セットアップ

```bash
# 1. リポジトリをクローン
git clone https://github.com/TheRealGo/CogniSync-Readiness.git
cd CogniSync-Readiness

# 2. direnv を有効化（初回のみ）
direnv allow

# 3. アプリ起動
pnpm install        # 依存関係のインストール
pnpm build          # ビルド
pnpm start          # サーバー起動 → http://localhost:3000
```

## PVT（精神運動覚醒検査）

覚醒度と持続的注意力を反応時間で測定します。

### 手順

1. **コンディション入力** — スライダーで主観的コンディション（1〜10）を設定
2. **テスト時間設定** — デフォルト3分（1〜10分で変更可能）
3. **「テスト開始」をクリック**
4. **「ここをクリックしてスタート」** でテスト画面に入る
5. 画面中央に `+` が表示されたら待機（2〜10秒のランダム間隔）
6. カウンター（`0000 ms`）が表示されたら **スペースキー** をできるだけ速く押す
7. 制限時間まで繰り返し → 自動で結果画面へ

### 操作キー

| キー | 操作 |
|------|------|
| Space | 反応（カウンター停止） |

### 結果の見方

| 指標 | 意味 |
|------|------|
| Mean RT | 平均反応時間（低いほど良い） |
| Minor Lapse | RT > 355ms の回数（注意の揺らぎ） |
| Major Lapse | RT > 500ms の回数（覚醒低下） |
| False Start | カウンター表示前に押してしまった回数 |
| Slowest 10% 平均 | 最も遅い上位10%の平均RT（疲労感に最も敏感） |

### 結果画面の色

| 色 | 意味 |
|----|------|
| 緑 | 正常（≤ 355ms） |
| 黄 | Minor Lapse（355ms超） |
| 赤 | Major Lapse（500ms超） |
| オレンジ | フォルススタート |

## Flanker Task（フランカー課題）

抑制制御とノイズフィルタリング能力を測定します。

### 手順

1. **コンディション入力** — スライダーで主観的コンディション（1〜10）を設定
2. **試行回数設定** — デフォルト40回（10〜80回で変更可能）
3. **「テスト開始」をクリック**
4. **「ここをクリックしてスタート」** でテスト画面に入る
5. `+` が表示されたら待機（0.5秒）
6. 5つの矢印が表示されたら、**真ん中の矢印の向き** だけを判断して対応するキーを押す

### 操作キー

| キー | 操作 |
|------|------|
| F | ◀ 左向きと判断 |
| J | ▶ 右向きと判断 |

### 刺激パターン

| パターン | 種類 | 正解 |
|----------|------|------|
| ◀◀◀◀◀ | 一致（Congruent） | F（左） |
| ▶▶▶▶▶ | 一致（Congruent） | J（右） |
| ▶▶◀▶▶ | 不一致（Incongruent） | F（左） |
| ◀◀▶◀◀ | 不一致（Incongruent） | J（右） |

### 結果の見方

| 指標 | 意味 |
|------|------|
| 正答率 | 正しく回答できた割合 |
| 干渉効果 | 不一致RT平均 − 一致RT平均（低いほど抑制制御が良好） |
| RT 標準偏差 | 反応時間のばらつき（低いほど安定） |
| 一致試行 平均RT | 一致パターンへの平均反応時間 |
| 不一致試行 平均RT | 不一致パターンへの平均反応時間 |

## 結果履歴

PVT / Flanker Task のタブで過去の記録を確認できます。

### トレンドチャート

セッションが2件以上あると、トレンドチャートが表示されます。

- **PVT**: Mean RT（赤線）と 主観スコア（青点線）の推移
- **Flanker**: 干渉効果（オレンジ線）と 主観スコア（青点線）の推移

### セッション管理

- 各セッションをクリックすると詳細メトリクスが展開される
- セッション個別の「削除」、または全記録の一括削除が可能

---

## 推奨ルーティン

毎日同じ時間帯に以下のフローで実施すると、コンディションの変化を追跡できます。

1. **主観評価**（10秒）— コンディションを1〜10で入力
2. **PVT**（3分）— 覚醒度チェック
3. **Flanker Task**（40試行 / 約2.5分）— 抑制機能チェック
4. **合計**: 約6分

> 3ヶ月間設定を固定して運用すると、酒・睡眠・ストレスなど日々の微細な変化が数値として浮かび上がります。
